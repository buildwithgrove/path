:::tip .values.yaml File

When running PATH in Tilt, an optional `./local/path/.values.yaml` file may be generated from the `values.tmpl.yaml` file.

To generate the `.values.yaml` file, run the following command:

```bash
make copy_values_yaml
```

Tilt will watch this file and automatically update the GUARD deployment with the new values on save.

:::

For example, the following values may be used to customize the GUARD Helm chart in two main ways:

1. Add or modify services in the `guard.services` array.

2. Add or modify API keys in the `guard.auth.apiKey.apiKeys` array.
<!-- TODO_DOCUMENT(@commoddity): Add a section describing the way GUARD authorizes requests using API keys in the Helm Charts docs and embed it in these docs. -->

```yaml
###########################################################################
# -------------------- GUARD Specific Configurations -------------------- #
###########################################################################
guard:
  # Add or modify services in the `guard.services` array.
  services:  
    - serviceId: F00C
      aliases:
        - eth
    - serviceId: F021
      aliases:
        - polygon

  auth:
    apiKey:
      # To disable API key authentication, set `enabled` to `false`.
      enabled: true
      # Add or modify API keys in the `guard.auth.apiKey.apiKeys` array.
      apiKeys:
        - test_api_key
```
